* Scope

This records the first TypeScript real-project syntax flow for RFC 011 using the
public =asp typescript= facade against =languages/typescript-lang-project-harness=.
It is evidence for the native-projection v1 lane only.  It does not claim cache hit replay, Julia cache payoff, arbitrary TypeScript/Python S-expression support, or CodeQL backend execution.

* Commands

The measured flow used:

#+begin_example
.bin/asp typescript search prime --view seeds languages/typescript-lang-project-harness
.bin/asp typescript query --selector src/parser/native_syntax/tree-sitter-query.ts --treesitter-query '(function_declaration name: (identifier) @function.name)' languages/typescript-lang-project-harness
.bin/asp typescript query --selector src/parser/native_syntax/tree-sitter-query.ts:202:221 --treesitter-query '(function_declaration name: (identifier) @function.name)' --code languages/typescript-lang-project-harness
.bin/asp typescript query --from-hook direct-source-read --selector src/parser/native_syntax/tree-sitter-query.ts:202:221 --code languages/typescript-lang-project-harness
#+end_example

The repeated warm syntax-frontier probes used the same selector and pattern as
the second command.

* Observations

- =search prime= surfaced =src/parser/native_syntax/tree-sitter-query.ts= as an
  owner without inlining source code.
- The syntax frontier returned compact capture/frontier output and located
  =renderTypeScriptTreeSitterQuery= at
  =src/parser/native_syntax/tree-sitter-query.ts:202=.  It did not emit raw code.
- Exact selector code output returned pure source stdout of 1037 bytes / 21
  lines.  Hook recovery returned pure source stdout of 1036 bytes / 20 lines.
  The source text is intentionally not copied into this evidence document.
- A temporary =PRJ_CACHE_HOME= measurement attempt failed with
  =provider-activation-missing= because facade activation is derived from the
  installed activation/cache home.  The accepted evidence therefore uses the
  real installed activation and claims only =warm-provider=.
- Warm-provider behavior was observed, but this is not a cache hit.  The
  native-projection v1 benchmark set is ready for review with the Rust evidence
  record, but cache replay remains a separate gate.
- RFC 011 Rust and TypeScript evidence records have been reviewed together in
  =docs/30-39-research/31.21-rfc011-native-projection-benchmark-review.org=.
