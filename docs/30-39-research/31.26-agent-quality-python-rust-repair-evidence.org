* Python repair path

The compact output exposed:

#+begin_example
[PY-AGENT-R010] Info: Function owns a broad linear algorithm surface
| Signals: control-flow.broad-linear-phase
#+end_example

Following that signal led to =run_protocol_cli= in
=src/python_lang_project_harness/_cli_protocol.py=.  The function mixed top-level
command routing, fast semantic search paths, harness fallback execution, and
error handling in one broad linear body.  The repair introduced named helpers
for =ast-patch=, fast protocol rendering, and harness fallback routing.  The
top-level function now owns only command dispatch and ValueError handling.

After the repair, =asp python check --changed .= returned clean output and the
full real-project JSON check reported zero quality findings.

* Rust repair path

The compact output exposed:

#+begin_example
Help: ... semantic_fact_owners ... Signals: native-idiom.manual-transform-loop.
#+end_example

Following that signal led to =semantic_fact_owners= in
=src/search/semantic_facts.rs=.  The function manually appended owner candidates
from two sources while tracking a =HashSet= of seen display paths.  The first
repair kept the same ordering and de-duplication key but expressed the pipeline
as =candidate_owners(...).take(...).chain(...).filter(...).collect()=.

The remaining Rust findings were in the same owner:

- an inline =#[cfg(test)]= module, moved to =tests/unit/search/semantic_facts.rs=
- =project_collection_field_owners= traversal scaffolding, split into directory
  scanning and path routing helpers
- =emit_collection_fields_from_syntax= and the public renderer's owner loop,
  rewritten to iterator pipelines

After the full repair, =asp rust check --changed .= returned =[ok] rust= and
the real-project JSON check reported zero findings.
