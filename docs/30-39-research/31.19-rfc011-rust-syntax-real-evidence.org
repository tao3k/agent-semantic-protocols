asp rust query --selector crates/agent-semantic-tree-sitter/src/query_syntax.rs:91:95 --treesitter-query '(function_item name: (identifier) @function.name)' --workspace . --code
asp rust query --from-hook direct-source-read --selector crates/agent-semantic-tree-sitter/src/query_syntax.rs:91:95 --code .

The repeated warm syntax-frontier probe used the same selector and pattern as the
second command.

* Observations

- Root-wide inline syntax frontier was a miss for this repository, while the
  repo-relative owner selector returned a non-miss frontier.
- Package-local locators from owner item output are useful for item navigation,
  but =--treesitter-query + --code= and hook recovery need the repo-relative
  selector form for stable exact code output.
- Exact selector code output and hook recovery both returned pure source stdout
  of 255 bytes / 5 lines.  The source text is intentionally not copied into this
  evidence document.
- Warm-provider behavior was observed, but this is not a cache hit.  The Rust
  lane remains paired with the TypeScript evidence record for benchmark review.
- RFC 011 Rust and TypeScript evidence records have been reviewed together in
  =docs/30-39-research/31.21-rfc011-native-projection-benchmark-review.org=.
