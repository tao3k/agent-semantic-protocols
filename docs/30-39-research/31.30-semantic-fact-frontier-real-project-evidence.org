fact frontier feedback loop.  The run uses this repository as the real project
and exercises the public =asp python= search facade, graph-turbo ranking, one
exact frontier read, and the focused frontier receipt schema validation test.

This record proves that the P1 receipt shape can describe an actual
agent-facing graph-turbo frontier loop.  It does not claim benchmark scoring,
learned rank weights, or cross-language receipt emission.

* Observations

- The search path now exercises Python graph-turbo directly.  Earlier fallback
  symptoms were fixed before this record by restoring the PPR result API and
  making =relationChannels= packet-visible.
- The compact frontier exposed exact selector actions.  The agent followed
  =H2.code= into
  =tests/unit/test_semantic_fact_frontier_receipt_schema.py:9-29= instead of
  falling back to raw source reads or broad file scans.
- The receipt fixture is intentionally separated from the inline schema unit
  cases.  The unit test validates the fixture catalog against
  =semantic-fact-frontier-receipt.v1.schema.json= and checks that recorded
  counts match the arrays in the receipt.
- Graph-turbo now has an explicit =asp-graph-turbo receipt= runtime capture
  command.  The command ranks a graph-turbo request, preserves node selector
  fields from the in-memory =GraphResult=, and emits the receipt schema from
  caller-supplied followed/read/test facts.
- ASP search now has an explicit =--frontier-receipt-out= runtime capture
  trigger for graph-turbo frontier search commands.  The protocol facade
  strips this flag before provider execution and rejects non graph-turbo
  renderer paths instead of preserving legacy =[search-*]= replay behavior.
- Graph-turbo now has an explicit =benchmark= command.  The command emits
  schema-owned latency and typed-path width metrics so future sandtables can
  compare path backend, pair count, candidate count, and fallback count without
  parsing debug text.
- Graph-turbo now has an explicit =sandtable-summary= command.  The command
  joins =semantic-graph-turbo-benchmark.v1= with a
  =semantic-fact-frontier-receipt.v1= receipt or fixture catalog into a single
  schema-owned comparison row.
- The offline benchmark report fixture now joins the task-session receipt,
  graph-turbo runtime receipt, ASP runtime frontier-only receipt, and ASP
  runtime followed/read/test receipt into one schema-owned table.  It is enough
  to gate first scoring decisions without parsing debug text; relation weight
  changes still need live sandtable scoring against this report.
