* Scope

This receipt fixes immutable artifact pointer compare-and-set behavior inside the
existing =agent-semantic-client-db= owner.  It does not introduce a parallel
storage crate or place domain dependencies below =languages/=.

* Fixed scenario tests

Command:

#+begin_src sh
direnv exec . cargo test -p agent-semantic-client-db artifact_pointer -- --nocapture
#+end_src

Result: =4 passed; 0 failed=.

The checked-in scenarios are:

- initial revision-zero apply, revision-one advance, stale conflict, and a
  successful advance after closing and reopening the database;
- sixteen independent Turso handles racing from revision zero, with exactly one
  =Applied= receipt and fifteen typed =Conflict= receipts;
- append-only failed-artifact evidence, duplicate attempt rejection, and proof
  that preserving a failed candidate does not advance the successful pointer.
- memory, topology, and coordination root authorities sharing the same pointer
  name remain isolated by their typed domain key and advance revisions
  independently.

The scenario owner is
=crates/agent-semantic-client-db/tests/unit/db/engine/artifact_pointer.rs=.
