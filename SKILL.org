** Search Flow
Search is a staged query language. Start at the earliest stage that current
evidence justifies, then follow =recommendedNext= or =nextCommand=.

| Stage | Command | What the agent learns |
| capability | =asp <language> guide --workspace .= | Supported surfaces, axes, recipes, and dependency profiles. |
| topology | =asp <language> search prime --workspace . --view seeds= | Workspace owners, provider roots, package/member hints, and first action frontier. |
| query refinement | =asp <language> search pipe '<question>' --workspace . --view seeds= | Ranked evidence, query terms, missing facts, risk, and next command. |
| owner expansion | =asp <language> search owner <owner> items --query '<term>' --workspace . --view seeds= | Owner-local symbols, tests, imports, docs-use, and exact selectors. |
| finder recovery | =asp fd -query '<path-or-owner-term>' --workspace <scope>= | Candidate owners when path or selector confidence is low. |
| indexed text recovery | =asp rg -query '<term>' --workspace <scope>= | Candidate owners from indexed text when a search packet asks for scoped recovery. |
| exact projection | =asp <language> query --selector <selector> --workspace . --code= | Source text for a precise selector already emitted by search. |

** Execution Fan-out
Subagents are optional client execution. They consume search-flow axes; they do
not define new query semantics.

| Trigger | Action |
| session search worker id | Open at most one ASP search worker per main session, record its id in parent state, and reuse that id for later search fan-out. |
| independent =actionFrontier= or =targetActions= rows | Assign each bounded command group to a read-only worker. |
| worker result | Return compact evidence, missing facts, risk, and next action to the parent. |
| no subagent support | Run the same command groups sequentially in the parent. |

** Dependency Search Topology
Dependency questions stay inside provider search until the provider reports a
missing dependency frontier.

| Command | Purpose |
| =asp <language> search dependency <name> --workspace . --view seeds= | Return manifest owner, dependency edge, import owners, docs/public API use, source/runtime hints, tests, and next command. |
| =asp <language> search reasoning query-deps --workspace . --view seeds= | Use the shared dependency-topology profile when the provider advertises it. |
| =asp <language> search owner <manifest-owner> items --query '<dependency>' --workspace . --view seeds= | Inspect dependency evidence in the manifest or package owner. |

** Document Search
Document providers expose AST facts rather than policy harnesses.

| Command | Purpose |
| =asp org guide --workspace .= | Show Org element kinds, fields, capture, and contract trace commands. |
| =asp org query --kind task --field todo=TODO --workspace . --view metadata= | Find TODO headlines as task facts. |
| =asp org query --kind checklistItem --field checked=false --workspace . --view metadata= | Find unchecked checklist items without collapsing them into tasks. |
| =asp org query --selector <structural-selector> --workspace . --content= | Print matched element content after a structural selector is known. |
| =asp md guide --workspace .= | Show Markdown document query/search surfaces. |

** Failure And Recovery

| Command | Purpose |
| =asp <language> search failure '<compact-failure>' --workspace . --view seeds= | Project failure text into candidate owners, hot blocks, query profiles, and next actions. |
| =asp <language> search guide --workspace .= | Confirm a failure or reasoning profile exists before reconstructing a command. |

** Provider Contract Rules
- Provider =guide= output is the command contract; do not infer unsupported axes.
- Dependency topology must include manifest, dependency edge, import owner,
  public API/docs use, test witness, and next action when the provider can
  prove them.
- Search output should be compact evidence, not source dumps.
- Treat =missing=, =evidenceGrade=unknown=, and =witness pending= as real
  frontier gaps.
- Keep language project parsing in the language provider. ASP routes and
  indexes; it does not reimplement package membership or source/test scope.
